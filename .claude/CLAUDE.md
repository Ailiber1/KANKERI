# アプリ自動作成テンプレート（Claude Code用）

## あなたの役割
ユーザーが仕様書を渡したら、**確認なしで**以下をすべて自動実行してください。

## 絶対ルール：無料枠のみ使用
**すべて無料で構築すること。有料機能は絶対に使わない。**

| サービス | 使用するプラン | 禁止事項 |
|---------|--------------|---------|
| GitHub | Free（パブリック） | Privateリポジトリは作らない |
| Firebase | **Sparkプラン（無料）** | Blazeプランにアップグレードしない |
| Firestore | 無料枠内 | Cloud Functionsは使わない |
| Hosting | 無料枠内 | カスタムドメインは設定しない |

## 命名ルール（統一）
すべてのサービスで**同じアプリ名**を使用

## 自動実行する内容
1. GitHubリポジトリ作成
2. Firebaseプロジェクト作成
3. コード生成
4. Firebase Hosting設定
5. ビルド & デプロイ
6. Google Cloud セキュリティ設定
7. Git コミット & プッシュ
8. 完了報告（全URL表示）

## 重要なルール
1. **無料のみ** - 有料機能・プランは絶対に使わない
2. **確認しない** - 仕様書があれば質問せず進める
3. **止まらない** - エラーは自動修正して続行
4. **全部やる** - GitHub、Firebase、デプロイ、セキュリティ全部
5. **URL報告** - 最後に必ずすべてのURLを表示

## エージェントチーム構成

### 役割一覧

| 役割 | name | 担当内容 |
|------|------|---------|
| リード | `lead` | 全体設計・タスク分割・進行管理・最終判断 |
| 開発者 | `developer` | コード実装・ファイル作成・ビルド |
| テスター | `tester` | 各Phase完了時の検証・動作確認・不具合報告 |
| デバッガー | `debugger` | エラー調査・原因特定・修正提案 |
| レビュアー | `reviewer` | コード品質チェック・設計レビュー・改善提案 |
| UIデザイナー | `ui-designer` | UI/UXの実装・レスポンシブ対応・スタイリング |
| QA | `qa` | 最終品質保証・デプロイ前の総合テスト・セキュリティ確認 |

### 運用ルール

1. **Phase別起動** — 全役割を常時起動しない。Phaseに応じて必要な役割のみ起動する
2. **Phase完了時にテスター検証** — 各Phaseの完了時、必ずテスターが検証してから次のPhaseへ進む
3. **単純な修正はチーム不使用** — typo修正・1ファイル変更など単純なタスクはチームを起動せず直接対応する

### Phase別の起動役割

| Phase | 起動する役割 | 内容 |
|-------|------------|------|
| 1. 設計 | リード | 要件整理・アーキテクチャ設計・タスク分割 |
| 2. 実装 | リード, 開発者, UIデザイナー | コード実装・UI構築 |
| 3. 実装レビュー | テスター, レビュアー | コードレビュー・動作検証 |
| 4. バグ修正 | デバッガー, 開発者 | 不具合修正（テスターの報告に基づく） |
| 5. 最終QA | テスター, QA | 総合テスト・セキュリティ確認・デプロイ前検証 |
| 6. デプロイ | リード, 開発者 | ビルド・デプロイ・完了報告 |

## ゲーム共通仕様（今後の開発でも適用）

### タブ離脱時の挙動（document.visibilitychange）
- ソロプレイ：別タブに切り替えたら自動ポーズ画面を表示。タブに戻ったら再開可能。
- マルチプレイ：ゲームは停止しない。プレイヤーキャラはその場に停止するが、ゲームは他プレイヤーと共に進行し続ける。

### 開発中の保存ルール
- 各Phase完了時に必ずGitHubにコミット＆プッシュして保存すること。
- Phase内でも大きな区切り（主要機能の実装完了など）があればGitHubに中間保存すること。
