# KANKERI（缶蹴り）ゲーム仕様書

## コンセプト
**"Run. Hide. Kick."** — **「缶ひとつ蹴れば、全員救える。」**
日本の伝統遊び「缶蹴り」を3Dリアルタイムマルチプレイゲームとして再現。
夜のネオン×日本の風景をテーマに、リアルな地形の中で隠れ、走り、缶を蹴る。

---

## 1. ゲームの基本ルール

### 缶蹴りの流れ
```
[1] 缶キック（タイミングゲージ）
    プレイヤーの1人が缶を蹴る
    ├── ゲージが上下に振動（パワーメーター）
    ├── タイミング良 → 缶が遠くに飛ぶ → 隠れ時間10〜15秒
    └── タイミング悪 → 缶が近くに落ちる → 隠れ時間3〜5秒
         ↓
[2] 隠れフェーズ
    鬼が缶を拾いに走る（缶の距離に比例した時間）
    その間にプレイヤーは地形を利用して隠れる
         ↓
[3] 鬼が缶を定位置に戻す → 探索フェーズ開始
    鬼がフィールドを歩き回りプレイヤーを探す
    ├── 鬼がプレイヤーを発見 → 缶の場所に戻って缶タッチ
    │   → 「見つけた！○○！」→ そのプレイヤーは捕獲（缶の近くに拘束）
    │
    │   ★ 鬼が缶に戻る途中でも、逃げ側は缶キック可能！
    │   ├── 見つかった本人が鬼より先に缶に辿り着く → 自力解放キック
    │   ├── 他の味方が隙を突いて缶に接近 → 救出キック
    │   └── 缶キック成功 → 捕獲された全員が解放 → [1]に戻る
    │       鬼の缶タッチ判定と逃げ側の缶キック判定は同時に走る
    │       → どちらが先に缶に到達するかの駆け引きが生まれる
    │
    └── 隠れ側プレイヤーが缶に接近してキック成功（探索中いつでも）
        → 捕獲された全員が解放 → [1]に戻る
         ↓
[4] 終了条件
    ├── 全員捕獲 → 鬼の勝ち
    └── 制限時間切れ → 逃げ側の勝ち（捕獲されていないプレイヤー全員勝利）
```

### 缶キック・タイミングゲージの仕様
```
[ゲージUI]
  ┌─────────────────────────────┐
  │  ▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░  │  ← ゲージが左右に振動
  │           ↑                  │
  │        PERFECT               │  ← 中央付近でタップ = 最大飛距離
  └─────────────────────────────┘

- ゲージは0%〜100%を往復（速度はステージで変化）
- PERFECT (80-100%): 缶が最大距離に飛ぶ → 隠れ時間15秒
- GREAT (60-79%): 中距離 → 隠れ時間10秒
- GOOD (30-59%): 近距離 → 隠れ時間6秒
- MISS (0-29%): 目の前に転がる → 隠れ時間3秒
- 缶を蹴り返す時（解放キック）もゲージ判定あり
  → PERFECT: 全員解放 + 缶が遠くに飛ぶ
  → MISS: 全員解放だが缶が近い → 鬼がすぐ戻ってくる
```

### 缶キックの接近判定とボタン操作
```
[缶キックの発動条件]
- プレイヤーが缶から2.5m以内に入ると「蹴る」ボタンが出現
  - PC: 画面中央に「Space: 缶を蹴る！」のプロンプト表示
  - モバイル: 画面右下に大きな「蹴る」ボタンが出現
- ボタンを押す → タイミングゲージが起動 → もう一度ボタンでゲージ確定＆キック

[鬼 vs プレイヤーの缶到達レース]
- 鬼が「見つけた！」宣言後、缶に戻る途中でも缶キック可能
- 判定の優先順位:
  1. プレイヤーが先に缶に到達＋蹴るボタン押下 → 解放キック成立
  2. 鬼が先に缶に到達＋缶タッチ → 捕獲成立
  → 完全に同時（0.1秒以内）の場合 → プレイヤー側（逃げ側有利）
- 鬼が缶に走っている間、鬼の移動速度は通常の120%（急いで戻る演出）
- ただし鬼は缶に戻る経路上のプレイヤーを無視する（缶タッチ優先）

[誰が蹴れるか]
- 捕獲されていないプレイヤー全員が蹴れる
  - 見つかった本人: 鬼より速く缶に辿り着けば自力キック
  - 別の味方: 鬼が缶から離れている隙に缶に接近してキック
- 捕獲されたプレイヤーは缶の近くに拘束されるため蹴れない
  （拘束位置は缶から3m → 蹴れる距離2.5mより遠い）

[缶キック中の無敵判定]
- ゲージが表示されている間（約1.5秒間）、鬼の缶タッチ判定は無効
- ゲージ確定してキックモーションが完了するまで保護される
- ただしゲージ表示前（ボタン押す前）に鬼が缶タッチすれば鬼が先
```

## 2. プレイモード

### 2.1 ストーリーモード（ソロ）
- プレイヤー（自分）+ 味方CPU（0〜3体）vs AI鬼
- ステージ1〜5を順にクリア（順序ロック）
- 制限時間内に生き残る or 缶を蹴って仲間を解放でクリア

#### ストーリーモード開始前の設定画面
```
┌─────────────────────────────────────┐
│         ストーリー設定               │
│                                     │
│  ● ステージ: [Stage 1 ▼]           │
│                                     │
│  ● 味方CPU人数:                     │
│    [0人] [1人] [2人] [3人]          │
│    ※ 0人 = 自分1人で挑戦           │
│    ※ 最大3人（自分+CPU3 = 4人）     │
│                                     │
│  ● CPUレベル:                       │
│    [弱] [中] [強]                   │
│                                     │
│  ● 鬼の強さ:                        │
│    ステージに応じて自動設定          │
│                                     │
│         [ゲーム開始]                 │
└─────────────────────────────────────┘
```

#### 味方CPUのレベル別挙動
| レベル | 隠れ行動 | 缶キック | 判断力 |
|--------|---------|---------|--------|
| **弱** | 近くの隠れ場所に隠れる。動かない。鬼が近くても逃げない | 缶キックしない（救出行動なし） | 同じ場所に隠れ続ける。覗きしない |
| **中** | やや遠くの隠れ場所を選ぶ。鬼が近づくと別の場所に移動 | たまに缶キックを試みる（成功率50%） | 鬼の位置を考慮して行動。たまに覗いて確認 |
| **強** | 最適な隠れ場所を選ぶ。鬼の視界を避けて移動 | 積極的に缶キックで救出（成功率80%） | 鬼の巡回パターンを読んで隙を突く。覗きで鬼の位置を把握してから行動 |

#### 味方CPUの行動パターン
```
[隠れフェーズ]
- 隠れ場所リストからレベルに応じた場所を選択
- 弱: 最も近い場所 / 中: ランダム / 強: 鬼から最も遠い場所

[探索フェーズ]
- 弱: 隠れ場所で静止。捕まるまで動かない
- 中: 鬼が10m以内に来たら別の隠れ場所に移動。缶キック成功率50%
- 強: 鬼の視界外を計算して移動。仲間が捕まったら缶キックを狙う

[缶キック判定]
- 中: 缶から15m以内 & 鬼が缶から20m以上離れている時のみ試行
- 強: 缶から20m以内 & 鬼が缶から10m以上離れていれば試行
       鬼の背後から回り込むルートを計算
```

### 2.2 スコアアタック（ソロ — ランキング対象）
```
1人 vs AI鬼、制限時間なし。捕まるまで生き残る。
生存時間がそのままランキングスコアになる。

[基本ルール]
- 味方CPUなし（自分1人）
- ステージ選択可能（クリア済みステージのみ）
- 制限時間なし → 捕まった時点で終了
- 缶キック成功 → 鬼がリセット（缶回収からやり直し）
  → 時間を稼ぐ手段として缶キックが重要

[鬼の増加（通常モードより攻撃的）]
  0:00〜  鬼1体
  1:00〜  鬼2体
  2:00〜  鬼3体
  3:00〜  鬼4体
  4:00〜  鬼5体（最大）
  → 通常モード(§3)より速いペースで増加

[キャンプ防止: 滞在検知システム]
- MAZE-HUNTと同じ方式:
  同じ場所（半径5m以内）に20秒以上留まると警告
  → 「位置が鬼に感知された！」通知
  → 最も近いAI鬼がプレイヤーの現在位置に向かって直行
  → プレイヤーは移動を強いられる
- 移動すれば（5m以上離れれば）滞在タイマーリセット
- 覗き中（ピーク）も滞在としてカウント（壁裏にずっと隠れ続けるのを防止）

[複数鬼の役割分担]
- 鬼が2体以上いる場合、自動的に役割が分かれる:
  ★ 缶番鬼（1体）: 缶から10m以内を巡回。缶キックを警戒
    - 缶を中心に円形パトロール
    - ランナーが缶に近づくと即追跡
  ★ 探索鬼（残り全員）: ステージ全体を巡回してプレイヤーを探す
    - 通常の巡回→追跡→缶タッチの行動パターン
    - 陽動にも反応する（ただし缶番鬼は陽動に反応しない）
- 缶番鬼が追跡で離れた場合、探索鬼の1体が缶番に切替（常に1体は缶の近く）

[ランキング記録]
- 生存時間: mm:ss.ms（分:秒.ミリ秒）
- ステージ別ランキング + 総合ランキング（全ステージの最長記録）
- ベスト記録のみ上書き（ranking-pattern.md準拠）

[設定画面]
┌─────────────────────────────────────┐
│       スコアアタック設定             │
│                                     │
│  ● ステージ: [Stage 1 ▼]           │
│    ※ クリア済みステージのみ選択可   │
│                                     │
│         [ゲーム開始]                 │
└─────────────────────────────────────┘
```

### 2.3 マルチプレイ（最大4人）

#### 個人戦（Free For All）
- 全プレイヤー（最大4人）が逃げ側、AI鬼が追う
- 最後まで捕まらなかったプレイヤーが勝利
- 缶キック解放は誰でも可能（自分が捕まっていなければ）
- AI鬼は時間経過で増加（§3準拠）
- 逃げ切り → 個人statsのescapeCountに加算（ランキングはスコアアタックのみ）

#### チーム戦（Team Battle）
```
鬼チーム vs 逃げチーム（最大4人を分割）。
鬼チームはプレイヤーが鬼を操作して探す（操作詳細は§9.2）。
ラウンド制で公平性を確保。

[チーム分割]
- 2人: 1v1（鬼1 vs ランナー1）
- 3人: 1v2（鬼1 vs ランナー2）→ 次ラウンドで交代
- 4人: 2v2（鬼2 vs ランナー2）

[ロビーでのチーム選択]
- ホストがチーム分けを決定（ドラッグ&ドロップ or ランダム）
- 各プレイヤーの selectedRole: "runner" | "oni" をFirebaseに保存
- 全員が「準備完了」→ ゲーム開始

[ラウンド制の流れ]
  ラウンド1: チームA=鬼, チームB=ランナー
    → 制限時間終了 or 全員捕獲 → スコア記録
    ↓
  インターバル（10秒、スコア表示）
    ↓
  ラウンド2: チームB=鬼, チームA=ランナー
    → 制限時間終了 or 全員捕獲 → スコア記録
    ↓
  最終スコア比較 → 勝敗決定

[スコア計算]
- 鬼チームのスコア: 全員捕獲までの時間（速いほど高得点）
  - 全員捕獲: 残り時間 × 100 ポイント
  - 時間切れ: 捕獲人数 × 500 ポイント
- 比較: ラウンド1とラウンド2のスコアを比較 → 高い方が勝利

[ラウンド間のリセット]
- 全プレイヤーの位置をリスポーン地点に戻す
- 捕獲状態をリセット
- 缶を定位置に戻す
- 鬼の疲労をリセット
```

#### 協力プレイ（Co-op）
- 全プレイヤー（最大4人）が逃げ側 vs AI鬼（複数体）
- 制限時間まで全員で生き残る
- 仲間が捕まったら缶キックで救出
- ステージ1〜5を順に攻略
- 全員捕獲 = 全滅 → 同ステージやり直し

## 3. 鬼の増加ルール（全モード共通）

### 時間経過による鬼の増加
```
各ステージには「初期鬼数」と「最大鬼数」がある。
時間経過に応じて鬼が追加スポーンする。

[増加タイミング]
- 制限時間の50%経過時: +1体
- 制限時間の75%経過時: +1体
- ただし最大鬼数を超えない

例: Stage 3（制限時間150秒、初期2体、最大4体）
  0秒〜:  鬼2体
  75秒〜: 鬼3体（+1体スポーン）
  112秒〜: 鬼4体（+1体スポーン）

[スポーン演出]
- 警告SE「新たな鬼が現れた！」
- フィールド端の暗がりから登場（プレイヤーの視界外）
- 3秒間の赤い光のスポーンエフェクト

[モード別の適用]
- ストーリー: ステージ定義に従う
- 個人戦(FFA): ステージ定義に従う
- チーム戦: 鬼チームがプレイヤー操作のため増加なし
- 協力プレイ: ステージ定義に従う（やや多め）
```

### ステージ別 鬼の数
| Stage | 初期 | 50%経過 | 75%経過 | 最大 |
|-------|------|---------|---------|------|
| **1** | 1体 | 1体（増加なし） | 2体 | **2体** |
| **2** | 1体 | 2体 | 2体 | **2体** |
| **3** | 2体 | 3体 | 4体 | **4体** |
| **4** | 2体 | 3体 | 4体 | **4体** |
| **5** | 3体 | 4体 | 5体 | **5体** |

※ 協力プレイではさらに+1体ずつ加算

```

AI鬼:  初期2体 → 最大4体
```

## 8. カメラシステム（三人称視点）

```
MAZE-HUNTの一人称視点(FPS)と異なり、KANKERIは三人称視点(TPS)を採用。
キャラクターの背中が常に見え、周囲の状況を把握しやすい。
缶蹴りの「隠れる→覗く→走る」という動きが視覚的に映える。

[基本カメラ]
- カメラ位置: キャラクターの後方3m、上方1.5m
- キャラクターの肩越しにやや右にオフセット（+0.5m）
- マウス / スワイプでキャラクターの周囲を360度回転可能
- FOV: 60度

[カメラの壁衝突]
- カメラが壁や建物にめり込まないようレイキャスト判定
- 壁に当たった場合、カメラがキャラクターに近づく（最短0.5m）
- 狭い裏路地では自動的にカメラが近くなる

[状態別カメラ演出]
| 状態 | カメラ距離 | 高さ | 特殊演出 |
|------|-----------|------|---------|
| 通常移動 | 3.0m | 1.5m | — |
| ダッシュ | 3.5m | 1.5m | FOV微増（65度）、画面端に速度線エフェクト |
| しゃがみ | 2.5m | 1.0m | カメラが低く寄る |
| 覗き（ピーク） | — | — | 下記「覗きカメラシステム」参照 |
| 缶キックゲージ中 | 2.0m | 1.0m | 缶にフォーカス、背景ぼかし風 |
| 捕獲された時 | 4.0m | 2.5m | 俯瞰気味、モノクロ風ポスト |
| 鬼が近い | 3.0m | 1.5m | 画面端が赤く脈動 |

[カメラとキャラクターの向き]
- キャラクターはカメラの向きではなく、移動方向を向く
- 静止中はカメラの正面方向を向く
- 覗き中は体の向きは変えず、首だけ覗き方向に回転
```

### 8.2 覗きカメラシステム（状況適応型）
```
覗き時にカメラを切り替えて画面奥の視認性を大幅に向上させる。
鬼の距離に応じて2段階のカメラモードを自動切替。

════════════════════════════════════════
 Phase 1: 肩越しズーム（鬼が15m以上離れている時）
════════════════════════════════════════

[カメラ移動]
- TPS位置（後方3m）→ キャラの肩横（後方0.8m、横0.4m）へ 0.3秒でlerp
- カメラの高さ: キャラの目線と同じ高さに下がる

[FOVズーム]
- 通常FOV 60° → 覗き中 40° に 0.3秒で補間
- 効果: 画面奥がズームアップされ、遠くの通路・鬼の姿が見やすくなる

[フォグ軽減]
- 通常のフォグ密度を覗き中は50%に軽減
- 効果: 暗闘でも奥まで見通せる（覗き = 目を凝らしている演出）

[視覚演出]
- 画面の上下に薄い黒帯（レターボックス風、高さ5%）
- 覗き方向と反対側の画面端が微かにぼける

════════════════════════════════════════
 Phase 2: 強制POV切替（鬼が15m以内に接近時）
════════════════════════════════════════

[トリガー]
- 覗き中（Phase 1）に鬼が15m以内に入ると自動発動
- プレイヤーの操作なし（演出として強制切替）

[カメラ移動]
- 肩越し位置 → キャラの目の位置（頭の中心）へ 0.2秒でlerp
- 完全一人称視点（自キャラ非表示、名前スプライトも非表示）

[FOV]
- 40°（Phase 1のまま維持 = ズーム状態で鬼が大きく見える）

[恐怖演出]
- 心拍SE: 鬼との距離に応じてテンポが変化
  - 15m: 60BPM（ドクン…ドクン…）
  - 10m: 90BPM（ドクンドクンドクン）
  - 5m以内: 120BPM（ドクドクドクドク）
- 画面端の赤いビネット: 鬼が近いほど濃くなる（opacity 0.1→0.5）
- カメラの微震え: 鬼が10m以内で微かに揺れる（振幅±0.005m）
- 鬼の目の黄色い光が暗闘でギラッと光る

[解除条件]
- 覗きキーを離す → 0.3秒でTPSに戻る
- 鬼が15m以上離れる → Phase 1に戻る（POV解除）

════════════════════════════════════════
 カメラ遷移まとめ
════════════════════════════════════════

  [TPS通常] ──覗きキー押下──→ [Phase 1: 肩越しズーム]
       ↑                           │
       │                    鬼が15m以内に接近
       │                           ↓
       │                    [Phase 2: 強制POV]
       │                           │
       └──覗きキー離す─────────────┘

[実装メモ]
- カメラ位置の補間: Vector3.lerp（位置）+ Quaternion.slerp（向き）
- FOV補間: THREE.MathUtils.lerp(currentFOV, targetFOV, 0.1)
- Phase切替はフレームごとに鬼との距離を計算して判定
- POV中のキャラメッシュ: visible = false（影も非表示）
- 壁の端からの覗き判定はレイキャストで壁端を検出
```

## 9. 棒人間キャラクター

### 9.1 デザインコンセプト
```
リアルな地形 × シンプルな棒人間 = KANKERIの独自スタイル。
パッケージ画像のシルエット（子どもたちの影）を棒人間で再現。
暗いステージでネオンに照らされた色付き棒人間が映える。

[カラールール]
- ランナー（プレイヤー/味方CPU）: 青色  #4488ff
- 鬼（AI / プレイヤー操作）:       赤色  #ff2222
- 捕獲されたランナー:              灰色  #666666（拘束中は色が抜ける）
- マルチプレイ時:
  - ランナー全員: 青  #4488ff（自分も他プレイヤーも同じ青）
  - 鬼全員:       赤  #ff2222（チーム戦で鬼が複数でも全員赤）
- 頭上表示（MAZE-HUNTと同じ方式）:
  - 全プレイヤーの頭上にユーザー名を常時表示
  - Canvas2Dでテキストを描画 → SpriteMaterialでビルボード表示
  - フォント: Share Tech Mono, 白文字, 黒縁取り（視認性確保）
  - 自分のキャラ: 名前の上に「▼」マーカー追加
  - 鬼の名前: 赤文字（チーム戦で鬼がプレイヤーの場合）
  - 距離に応じて名前サイズが自然に縮小（3Dスプライトなので自動）
```

### 9.2 棒人間の構造（プロシージャル生成）
```
GLBモデル不使用。Three.jsのプリミティブで組み立てる。

[パーツ構成]
  ● 頭:   SphereGeometry(0.15, 16, 16)
  │ 首:   なし（頭を胴体上部に直接配置）
  ┃ 胴体: CylinderGeometry(0.06, 0.06, 0.5, 8)
  ┣━ 左腕: CylinderGeometry(0.03, 0.03, 0.4, 8) × 上腕 + 前腕
  ┣━ 右腕: 同上
  ┣━ 左脚: CylinderGeometry(0.04, 0.04, 0.45, 8) × 太もも + すね
  ┗━ 右脚: 同上

[マテリアル]
- MeshStandardMaterial({
    color: キャラカラー,
    emissive: キャラカラー,
    emissiveIntensity: 0.3,    // 暗闘でも自身が少し光る
    roughness: 0.7,
    metalness: 0.1
  })
- 鬼は emissiveIntensity: 0.5（より赤く光る = 脅威感）

[サイズ]
- 全身高さ: 約1.7m（リアルな人間スケール）
- 頭の半径: 0.15m
- 胴体長: 0.5m
- 腕長: 0.4m × 2（上腕+前腕）
- 脚長: 0.45m × 2（太もも+すね）

[関節（ピボットポイント）]
- 肩: 胴体上部の左右
- 肘: 上腕と前腕の接続点
- 股関節: 胴体下部の左右
- 膝: 太ももとすねの接続点
- 首: 胴体上部（頭の回転用）
- 各関節はObject3Dの階層構造（parent-child）で表現
  → parent.rotation を変えるだけで子パーツが追従
```

### 9.3 棒人間のアニメーション（プロシージャル）
```
全アニメーションは毎フレーム関節角度を計算するプロシージャル方式。
AnimationMixer, GLBアニメーション不使用。

[待機（Idle）]
- 軽い上下のバウンス（Y: ±0.02m, 周期2秒）
- 腕は自然に下げた状態

[歩行（Walk）]
- 左右の脚を交互に振る（股関節: ±20度, 周期0.8秒）
- 腕は脚と逆位相で振る（肩: ±15度）
- 膝は脚が後ろに行く時に少し曲げる

[走行（Run / ダッシュ）]
- 脚の振り幅を大きく（股関節: ±35度, 周期0.4秒）
- 腕の振り幅も大きく（肩: ±30度）
- 膝の曲がりも大きく
- 胴体がやや前傾（X回転: 10度）

[しゃがみ（Crouch）]
- 膝を大きく曲げる（膝: 90度）
- 胴体が低くなる（Y位置: -0.4m）
- 頭の位置が通常の60%の高さに

[しゃがみ歩き]
- しゃがみ姿勢のまま脚を小さく振る（振り幅: ±10度, 周期1.0秒）

[覗き（Peek）]
- 胴体を左or右に傾ける（Z回転: ±15度）
- 頭が壁の端から出る
- 脚は動かない

[缶蹴りモーション]
- 片脚を後ろに引く（0.2秒）→ 前に大きく振り出す（0.1秒）
- 蹴り足が缶に当たるタイミングで缶が飛ぶ

[叩くモーション]
- 片腕を振り上げる（0.2秒）→ 前方に振り下ろす（0.1秒）
- 対象物に当たるタイミングで音SE

[捕獲された時]
- 座り込みポーズ（膝を抱える）
- 色が灰色に変化（0.3秒でフェード）

[鬼の追跡時]
- 走行と同じだが腕を前に伸ばす（追いかける感）
- emissiveIntensityが0.5→0.8にUP（より赤く光る）
```

### 11.2 鬼プレイヤー操作（チーム戦専用）
```

════════════════════════════════════════

════════════════════════════════════════

[基本移動]
- ランナーと同じ操作（WASD/矢印 + マウス + Shift=ダッシュ）
- 鬼のダッシュはスタミナ無限（ただし疲労システム§9.4は適用）
- しゃがみ不可（鬼にしゃがむ必要がない）
- 覗き不可（鬼は堂々と探す）

[ロックオン（発見宣言）— ★最重要メカニクス]
- 鬼プレイヤーの画面中央に常時「照準レティクル」を表示
  （薄い赤い円、ランナーに重なると赤く光る）

  ┌─────────────────────────┐

  PC: 左クリック / モバイル: 画面右の「発見！」ボタン

[ロックオン判定条件（不正防止）]

  1. 鬼とランナーの距離が20m以内
  2. 鬼の視線方向（rotY）とランナーへの角度差が±30度以内
  3. 鬼とランナーの間にレイキャスト → 遮蔽物がない（壁越し発見を防止）
  4. ランナーが覗き中の場合: 距離8m以内に制限（§9.7覗き仕様準拠）
- 条件を満たさない場合 → ロックオン失敗（SEなし、画面に赤い×表示）

[発見宣言の流れ]
  ロックオン成功
    ↓
  「見つけた！○○！」が全プレイヤーに通知

    ↓

    ↓
  Space / タップ → 缶タッチ → 対象ランナー捕獲確定
  （缶タッチ後2秒間の硬直 = AI鬼と同じ）

[缶タッチ（鬼の缶踏み）]
- 操作: Space（PC）/ 画面の「缶タッチ」ボタン（モバイル）

- 缶タッチ後2秒間硬直（名前コール演出）

[ロックオン中に缶を蹴られた場合]

- 鬼は蹴り飛ばされた缶を回収しに行く必要がある

[缶の回収]

- 缶を持った状態で定位置に戻る → 定位置近くでSpace → 缶を置く
```

### 11.3 鬼のAI（視界ベース探索 — ストーリー/FFA/協力/スコアアタック用）
```
[巡回モード]
- 定められたウェイポイントを巡回
- 視界角度: 前方120度、距離20m
- 視界内にプレイヤー → [追跡モード]に切替

[追跡モード]
- 最後にプレイヤーを見た位置に向かう
- 到着しても見つからない → [巡回モード]に戻る
- プレイヤーを視認し続けている → [缶タッチモード]に切替

[缶タッチモード]（★鬼が缶に戻る途中）
- 発見したプレイヤーの名前を記憶して缶に向かって走る（速度120%）
- この間、経路上のプレイヤーは無視する（缶タッチ優先）
- ★ 逃げ側のプレイヤーが先に缶を蹴った場合:
  → 缶タッチ失敗 → [缶回収モード]に移行（蹴り飛ばされた缶を回収）
  → 記憶していた対象プレイヤーの捕獲もキャンセル
- 鬼が先に缶に到達 + 缶タッチ → 捕獲確定
  → 対象プレイヤーは缶の近く（3m）に拘束される
- 缶タッチ時に2秒間の硬直（名前コール演出）
  → この間に別のプレイヤーが缶に接近するチャンスが生まれる

[陽動反応モード]
- 環境音（プレイヤーが物を叩いた音）を検知 → 音の発生源に向かう
- 音源に到達して3秒間周囲を見回す → 誰もいなければ[巡回モード]に戻る
- 陽動中は視界判定が音源方向に集中（背後の警戒が甘くなる）
- ただし陽動中にプレイヤーを視認した場合は即[追跡モード]に切替

[缶回収モード]
- 缶が蹴り飛ばされた → 缶の位置に走る
- 缶を拾う → 定位置に戻す → [巡回モード]に復帰

════════════════════════════════════════
 キャンプ防止: 滞在検知システム（全モード共通）
════════════════════════════════════════
MAZE-HUNTと同じ方式。同じ場所に留まり続けるプレイヤーを鬼に通報する。

[滞在検知ルール]
- プレイヤーが半径5m以内に20秒以上留まると「滞在警告」発動
- 警告順序:
  15秒: 画面に「⚠ 長時間の滞在は危険です」テキスト表示（自分だけに見える）
  20秒: 「位置が鬼に感知された！」通知 + 警告SE
        → 最も近いAI鬼（探索鬼）がプレイヤーの正確な位置に直行
- 5m以上移動すれば滞在タイマーリセット
- 覗き中（ピーク）も滞在としてカウント

[滞在検知の例外]
- 隠れフェーズ中（まだ鬼が探していない間）はカウントしない
- 捕獲された状態（拘束中）はカウントしない

════════════════════════════════════════
 複数鬼の役割分担（鬼が2体以上いる時）
════════════════════════════════════════
鬼が2体以上いる場合、自動的に「缶番」と「探索」に役割が分かれる。
缶キックされるのを防ぎつつ、プレイヤーを見つけに行く連携を実現。

[役割の割り当て]
  ★ 缶番鬼（常に1体）:
    - 缶から10m以内を円形にパトロール
    - ランナーが缶から15m以内に入ると警戒度UP → 缶の近くで待ち伏せ
    - ランナーが缶に近づくと即[追跡モード]で阻止
    - 陽動（物を叩く音）には反応しない（缶の守りを優先）
    - 缶番鬼が追跡で缶から離れた場合 → 探索鬼の1体が缶番に自動切替

  ★ 探索鬼（残り全員）:
    - ステージ全体のウェイポイントを巡回
    - 通常の巡回→追跡→缶タッチの行動パターン
    - 陽動にも反応する
    - 滞在検知で通報されたプレイヤーの位置にも向かう

[役割切替のルール]
- 缶番鬼が追跡で缶から20m以上離れた → 最も缶に近い探索鬼が缶番に昇格
- 缶番鬼がプレイヤーを捕獲完了 → 缶の近くに戻って缶番再開
- 缶がキックされた → 缶番鬼が缶回収に向かう。その間は探索鬼が缶番を代行
- 探索鬼が全員追跡中 → 缶番鬼は缶の守りを維持（缶が無防備にならない）
```

### 11.4 鬼の疲労システム
```
鬼にもスタミナ（体力）があり、走り続けると疲労する。
プレイヤーが粘り強く逃げ回ることで鬼の性能が低下し、逆転のチャンスが生まれる。

[疲労ゲージ]
- 鬼1体ごとに疲労ゲージ（内部値 0〜100）を持つ
- プレイヤーからは見えない（鬼の挙動の変化で推測する）

[疲労の蓄積]
- 追跡モード中: +3/秒（走って追いかけると疲れる）
- 缶タッチモードで走行中: +4/秒（全力ダッシュ）
- 巡回モード中: -2/秒（歩いているので回復）
- 缶回収後の待機: -5/秒（缶を定位置に戻した後に少し休む）

[疲労段階と影響]
| 疲労値 | 状態 | 移動速度 | 視界距離 | 演出 |
|--------|------|---------|---------|------|
| 0〜30 | 通常 | 100% | 20m | — |
| 31〜60 | 疲れ気味 | 90% | 18m | 息切れSE（小） |
| 61〜80 | 疲労 | 80% | 15m | 息切れSE（大）、走りが遅い |
| 81〜100 | 消耗 | 70% | 12m | 肩で息をするアニメ、足を止めて3秒休む |

[消耗状態の休憩]
- 疲労が100に達すると強制的に3秒間立ち止まる
- 休憩中は移動不可・缶タッチ不可
- 休憩後に疲労が60まで回復 → [疲れ気味]状態から再開
- 鬼が立ち止まっている間は缶キックの大チャンス

[ステージ別の疲労速度]
- Stage 1〜2: 蓄積が遅い（初心者向け、鬼はあまり疲れない）
- Stage 3〜4: 標準
- Stage 5: 鬼が多いが個々の体力は標準（数で補う設計）
```

```
フィールドの特定のオブジェクトに近づくと「叩く」ボタンが出現。
叩くと音が鳴り、鬼の注意をその方向に引きつけられる。
缶蹴りの「音で鬼の気を逸らす」駆け引きを再現。

[操作]
- PC: Spaceキー（叩ける物の近く 1.5m以内で表示。缶が近くにない時のみ）

[叩ける物とステージ別一覧]
| Stage | 叩けるオブジェクト | 音の種類 | 鬼の反応距離 |
|-------|-------------------|---------|-------------|
| 1 | ブランコの鎖 | ガシャン（金属） | 25m |
| 1 | ゴミ箱 | ガンッ（鈍い音） | 20m |
| 2 | 賽銭箱の鈴 | リーン（鈴の音） | 30m |
| 2 | 石灯籠 | コンッ（石の音） | 15m |
| 3 | 自販機 | ガコン（缶が落ちる音） | 25m |
| 3 | 屋台のカウンター | バンッ（木を叩く音） | 20m |
| 3 | シャッター | ガラガラ（大きい音） | 30m |
| 4 | 教室のドア | バタン（ドアの音） | 20m |
| 4 | 鉄棒 | キーン（金属の響き） | 25m |
| 4 | 体育倉庫の壁 | ドンッ（壁を叩く音） | 20m |
| 5 | 屋台の鍋 | カーン（金属） | 25m |
| 5 | 橋の手すり | ガンガン（金属の響き） | 30m |

[鬼AIへの影響]
- 音の発生源が鬼の反応距離内 → 鬼が[陽動反応モード]に切替
- 鬼は音源に向かって移動（巡回や追跡を中断）
```

### 11.7 壁覗き（ピーク/リーン）
```

[発動条件]
- 壁・遮蔽物（建物、木、遊具、屋台等）の近く（1.5m以内）にいること
- 遮蔽物との距離・角度から「隠れ状態」と判定された時に覗きボタンが出現

[操作]
- PC: Q（左覗き）/ E（右覗き）— 押している間だけ覗く
- モバイル: 画面に覗きボタン（左👁 / 右👁）が出現 — タップ中覗く
- ボタンを離すと即座に元の隠れ位置に戻る

[カメラ挙動]
- 覗き方向にカメラが横にオフセット（±0.6m）
- カメラの高さはやや上がる（壁の角から顔を出す感じ）
- 覗き中はカメラ回転（視点操作）も可能
- スムーズに遷移（0.15秒でオフセット完了）

[キャラクター挙動]
- 覗き中はキャラモデルの上半身が傾く（左右に体を傾けるアニメーション）
- 頭と肩が遮蔽物の端からわずかに出る → 鬼から見える可能性あり
- 足は動かない（移動不可）

[鬼AIとの関係]
- 覗き中のプレイヤーの「頭」部分は鬼の視界判定に含まれる
- ただし露出面積が小さいため、鬼の発見判定に距離補正あり:
  - 通常視認距離: 20m → 覗き中: 8m（40%に低下）
  - 覗きは短時間なら安全、長時間は危険
- 鬼が「こちらを向いている」時に覗くと見つかりやすい

[マルチプレイ同期]
- playerPositions に peekDirection: "none" | "left" | "right" を追加
- 他プレイヤーから見ると壁から顔が出ているのが見える
```

```
- 各キャラのx座標を毎フレーム更新
- ランナーの速度: 1.5〜2.0 px/frame（個体差あり）
```

## 13. 操作（MAZE-HUNT準拠 + KANKERI追加アクション）

### 操作設計の方針
```
[初心者向け3つの原則]
1. 常時キーは3つだけ: 移動 + カメラ + ダッシュ（これだけで遊べる）
2. 状況キーは画面に表示: 近づくとUIガイドが出る（覚えなくてOK）
3. 片手で完結: 左手WASD周辺に全キー集約、右手はマウスのみ
```

### 基本操作（常時使用 — 最初に覚える3つ）
```
[PC]
- WASD / ←↑→↓（矢印キー）: 移動（どちらでも操作可能）
- マウス: カメラ回転（三人称視点でキャラの周囲を回転）
- Shift: ダッシュ（長押し、スタミナ消費）

[モバイル]
- 左下: 3Dバーチャルジョイスティック（移動）— MAZE-HUNTと同じ実装
- 右半分: スワイプでカメラ回転（三人称視点）
- 右下: ダッシュボタン（常時表示、長押し）
```

### 状況アクション（近づくと画面にキーガイドが表示される）
```
[PC — 画面にプロンプトが出るので覚えなくてOK]
┌─────────────────────────────────────────────────────────┐
│  キー    アクション    発動条件           画面表示       │
│─────────────────────────────────────────────────────────│
│  Space   缶を蹴る     缶から2.5m以内     「Space: 蹴る！」│
│  Space   物を叩く     叩ける物から1.5m   「Space: 叩く」  │
│  Q / E   左/右覗き    壁の近く1.5m       「Q◁ 覗き ▷E」 │
│  C       しゃがむ     いつでも           （HUDに常時表示）│
└─────────────────────────────────────────────────────────┘

★ 缶を蹴る・物を叩く を同じSpaceキーに統一（状況で自動切替）
  - 缶の近く → Space = 缶キック（最優先）
  - 叩ける物の近く → Space = 叩く
  - 両方の範囲内 → 缶キック優先（缶蹴りが最重要アクション）
  → 初心者は「近づいてSpaceを押す」だけで全部できる

★ 覗き（Q/E）は壁の近くでのみUIガイドが出現
  → 使わなくてもゲームは成立する（上級テクニック）

[モバイル — 状況に応じてボタンが自動出現]
- 「蹴る」ボタン — 缶の近くにいる時のみ出現（大きく目立つ）
- 「叩く」ボタン — 叩ける物の近くにいる時のみ出現
- 「しゃがむ」ボタン — 常時表示（トグル）
- [◁覗き] [覗き▷] — 壁の近くにいる時のみ左右に出現（タップ中のみ）

[キーガイドUI]
- 画面下部中央に、状況に応じてキーヒントがフェードイン表示
- 例: 缶に近づく → 「Space: 缶を蹴る！」がふわっと出現
- 3秒で自動フェードアウト（邪魔にならない）
- 設定でON/OFF可能（慣れたらOFFにできる）
```

### PC操作まとめ（全キー一覧）
```
┌───────────────────────────────────────┐
│    [Q覗き] [W前] [E覗き] [R]         │
│    [A左]   [S後] [D右]              │
│    [Shift=ダッシュ]  [C=しゃがむ]    │
│    [      Space=蹴る/叩く      ]     │
│                                       │
│  ★ 覚えるキーは実質4つだけ:          │
│    移動(WASD) + Shift + Space + C    │
│    Q/Eは上級者向け（なくても遊べる） │
└───────────────────────────────────────┘
```

## 14. マルチプレイ同期・バグ防止設計

### MAZE-HUNTの教訓
```
MAZE-HUNTで多発したマルチプレイバグのパターンと、KANKERIでの対策。
すべての判定をホスト権限で行い、クライアント間の不整合を防ぐ。
```

### 13.1 ホスト権限モデル（Authority）
```
[原則: ゲームの状態変更はすべてホストが行う]

クライアントが直接書き込めるデータ:
  ✅ 自分の位置・向き（playerPositions/{自分のindex}）
  ✅ チャットメッセージ（chat/）
  ✅ ロックオンリクエスト（lockOn/{自分のindex}、ただし validated=false）

ホストのみが書き込めるデータ:
  🔒 capturedPlayers/（捕獲判定）
  🔒 canState/（缶の状態変更）
  🔒 rooms/{roomId}/status（フェーズ遷移）
  🔒 oniPositions/（AI鬼の位置）
  🔒 lockOn/{index}/validated（ロックオン検証結果）
  🔒 teamBattle/（ラウンド管理、スコア）

[ホストが切断した場合]
- 次にplayerIndex が若いプレイヤーが新ホストに昇格
- Firebase の rooms/{roomId}/host を更新
- 新ホストは全プレイヤーの最新状態をFirebaseから再読込
- ホスト移行中（最大2秒）はゲームを一時停止（全員に「同期中...」表示）
```

### 13.2 判定の同期タイミング
```
[捕獲判定（AI鬼）]
1. ホストがAI鬼の視界判定を毎フレーム実行
2. 視界内にランナー発見 → ホストがstateを"chasing"に変更
3. AI鬼が缶に到達 → ホストが capturedPlayers/ に書き込み
4. 全クライアントがcapturedPlayersの変更をonValue()で検知
5. 捕獲されたプレイヤーのクライアントで拘束UIを表示

[捕獲判定（鬼プレイヤー — チーム戦）]
1. 鬼プレイヤーがロックオンボタン → lockOn/{index}/targetIndex を書き込み
2. ホストが検証（距離20m以内 + 視線角度±30度 + レイキャスト遮蔽なし）
3. 検証OK → validated=true + 全員に「見つけた！○○！」通知
4. 検証NG → validated=false + 鬼プレイヤーに×表示
5. 鬼が缶タッチ → ホストが缶との距離を検証 → capturedPlayers/ 更新

[缶キック判定]
1. ランナーが缶の近く（2.5m）でSpaceを押す → キックリクエストをFirebaseに送信
2. ホストがランナーと缶の距離を検証（2.5m以内か）
3. 検証OK → タイミングゲージ開始（ゲージ結果もホストに送信）
4. ゲージ確定 → ホストが canState/status="kicked" + canPosition更新
5. 全クライアントで缶の飛ぶアニメーション + 捕獲者の解放

[同時判定（缶タッチ vs 缶キック）]
- ホストのタイムスタンプで先着判定
- 0.1秒以内の同時 → ランナー優先（逃げ側有利ルール §1準拠）
- ゲージ表示中（1.5秒）の鬼の缶タッチは無効（§1準拠）
```

### 13.3 位置同期と補間
```
[位置データの送信頻度]
- 各クライアント: 100msごとに自分の位置をFirebaseに書き込み
  （playerPositions/{index}/x, y, z, rotY, t）
- 移動していない時: 500msごとに間引き（Firebase書込み節約）

[他プレイヤーの位置補間]
- 受信した位置データは100ms遅延で補間表示（lerp）
- 急激なワープ（前回位置から10m以上離れた場合）→ テレポート表示
- 補間中のプレイヤーに対するロックオン判定は、補間後の表示位置ではなく
  Firebaseの最新位置データで行う（見た目の遅延でロックオンが外れない）

[切断プレイヤーの処理]
- 3秒以上位置更新がない → 「接続不安定」表示
- 10秒以上更新なし → 切断扱い
  - ランナー: その場に立ったまま残る（鬼はロックオン可能）
  - 鬼プレイヤー: ホストがAI鬼に自動切替（巡回モードで代行）
  - 30秒以内に再接続 → 最後の位置から復帰
  - 30秒超 → ルームから退出扱い
```

### 13.4 チーム戦のラウンド同期
```
[ラウンド遷移の流れ]
1. ラウンド1終了 → ホストが round1Score を計算・書き込み
2. ホストが status="roundInterval" に変更
3. 全クライアントがインターバル画面を表示（10秒カウントダウン）
4. インターバル中にホストがチーム入替を実行:
   - teamA ↔ teamB のロール(oni/runner)を入替
   - 全プレイヤーの位置をリスポーン地点にリセット
   - canState をリセット
   - capturedPlayers をクリア
5. カウントダウン終了 → ホストが status="kickPhase" に変更
6. ラウンド2開始

[ロール変更時の注意]
- 鬼→ランナーへの切替: カメラを三人称に変更、レティクル非表示、HUD切替
- ランナー→鬼への切替: レティクル表示、鬼専用HUDに切替、スタミナバー→疲労バー
- 切替時にキャラモデルの入替アニメーション（1秒フェード）
```

### 13.5 バグ予防チェックリスト
```
[実装時に必ず確認する項目]

□ 全クライアントで同じゲーム状態が表示されているか
  → ホストの状態変更後、各クライアントのonValue()で正しく反映されるかテスト

□ ホスト切断時にゲームが止まらないか
  → ホスト移行ロジックを2人・3人・4人それぞれでテスト

□ 缶キックと缶タッチが同時に発生した場合の処理
  → ホスト側で0.1秒ルールが正しく動作するかテスト

□ ロックオンの不正防止が機能しているか
  → 壁越しロックオン、距離外ロックオン、視線外ロックオンをそれぞれテスト

□ 位置補間がカクつかないか
  → 高遅延環境（500ms）でもゲームが成立するかテスト

□ ラウンド切替時に状態がリセットされるか
  → 捕獲状態、缶位置、疲労値、ロックオン状態が全てクリアされるか

□ 3人プレイ時のチーム分け（1v2）が正しく動作するか
  → 鬼1人 vs ランナー2人でゲームが成立するか

□ フェーズ遷移（kick→hide→seek）が全クライアントで同期するか
  → 1人だけ違うフェーズにいる状態が起きないか

□ ゲスト（未登録ユーザー）がマルチプレイに参加した場合
  → ランキング登録をスキップするだけで、ゲーム自体は正常に動作するか
```
